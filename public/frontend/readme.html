Rails Tweet Setiment

1. Introduzione

"Rails Tweet Sentiment " è una web app che nasce con l'idea di effettuare Sentiment Analysis su tweet.

Lo scopo di effettuare l'analisi dei tweet è quello di "decifrare" il possibile Trend a cui un Tag o una Persona possono essere soggetti. 

L'Analisi del Sentiment è un analisi di tipo statistico volta a fotografare il pensiero degli utenti rispetto ad un Brand.

2. Manuale d'uso

2.1 Login

Essendo il sistema esposto pubblicamente in Internet e utilizzando librerie esterne (elencate di seguito) si è reso necessarie garantire la sicurezza dell'applicativo.

Il sistema utilizza una procedura di autenticazione basata su token

2.2 Ricerca

Una volta superata l'autenticazione, è possibile effettuare ricerche in 3 diverse modalità:

  • Tramite #TAG (esempio: #love, #java, #rails, ....)
  • Tramite @USER (esempio: @BarakObama, @ketyperry, @twitter, ...)
  • Tramite TESTO (esempio: developer, love, italy, ....)


NB: la ricercare può resituire al massimo 100 risultati, non è possibile averne di più perchè non si è a disposizione di un accaount premium di Indico (vedi sotto).

2.3 Visualizzazione Risultati

La visualizzazione dei risultati avviene nel seguente modo:

Sentimento Medio


 Mostra la media dei "sentiment" dei vari tweet. Se il valore è minore di 40% allora il sentimento è NEGATIVO, se compreso tra 40% e 60% è NEUTRO, se maggiore di 60% è POSITIVO.

Trend

 Viene mostrato un grafico formato dal "sentiment" dei singoli tweet. Cliccando sul tweet (il punto nel grafico)  viene visualizzata una finestra modale contente l IFRAME del tweet stesso 

Location of twitter's user

 Mostra una mappa in scala mondiale le nazionalità e il relativo conteggio degli utenti che hanno effettuato  tweet secondo il criterio di ricerca.

Tag Cloud

 Mostra una "Tag Cloud" contenente i relativi "tag" associati al criterio di ricerca

Keywords

 Mostra una "Tag Cloud" contenente le relative "keywords" associate al criterio di ricerca

JSON

 JSON contenete il "dataSource" prodotto dal server e sul quale sono stati prodotti i grafici.

Info

 Pannello riassuntivo sullo stato mensile dell'utenza 

3. Teconologie usate

3.1 Backend

  • Ruby On Rails: Open source web framework bsato su Ruby (http://rubyonrails.org):
  • MySQL: DBMS famoso per stabilità e performance (https://www.mysql.it).

3.2 Servizi di terze parti

  • Twitter: Vengono utilizzate le API pubbliche esposte da Twitter (https://dev.twitter.com, https://github.com/sferik/twitter):
  • Indico: Sistema che offre servizi di machine-learning (https://www.indico.io).  NB: Viene utilizzata una utenza FREE, per cui si ha una limitazione dei servizi, MAX 50.000 richieste mensili e tempi di risposta più lenti.
  • Geocoder: Utilizzato per la geolocalizzazione (https://developers.google.com, https://github.com/alexreisner/geocoder)

3.3 Frontend

  • JQuery: Semplice, Potente e veloce libreria JS (https://jquery.com).
  • Twitter Bootstrap: Interfaccia grafica contenente HTML,CSS e JS (http://getbootstrap.com).
  • JQuery UI: Libreria JQuery per effetti grafici (https://jqueryui.com).
  • C3: Librereia per disegnare Grafici, implementa d3, (http://c3js.org).
  • jVectorMap: Libreria usata per visualizzare Mappe (http://jvectormap.com).

4. API REST

Il sistema esponi API REST:


Per effettuare una ricerca:

curl -X POST --data "text='TEXT-TO-SEARCH'" http://51.254.200.28/api/v1/search -H 'Authorization: Token token="TOKEN"'


Per ottenere Info su utenza:

curl  http://51.254.200.28/api/v1/info -H 'Authorization: Token token="TOKEN"'
